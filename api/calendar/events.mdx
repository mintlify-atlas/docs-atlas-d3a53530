# Calendar Events API

Manage calendar events for project activities including maintenance schedules, equipment installations, removals, and inspections. Events can track resources, assign technicians, and monitor completion status.

## Create Calendar Event

<CodeGroup>
```bash POST /api/calendar/events/
curl -X POST https://api.mantis.com/api/calendar/events/ \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": 789,
    "title": "Equipment Maintenance",
    "description": "Scheduled maintenance for field equipment",
    "event_type": "MAINTENANCE",
    "priority": "HIGH",
    "start_date": "2026-03-20",
    "end_date": "2026-03-22",
    "start_time": "08:00",
    "end_time": "17:00",
    "responsible_technical_id": 123,
    "details": [
      {
        "resource_item_id": 101,
        "description": "Replace oil filter",
        "cost": 150.00
      }
    ]
  }'
```
</CodeGroup>

Create a new calendar event with resource details and assignments.

### Request Body

</ParamField>

<ParamField body="title" type="string" required>

</ParamField>

</ParamField>

<ParamField body="end_date" type="string">

</ParamField>

</ParamField>

<ParamField body="end_time" type="string">
  Event end time (HH:MM or HH:MM:SS)
</ParamField>

<ParamField body="responsible_technical_id" type="integer">
  ID of technical staff member responsible for this event
</ParamField>

<ParamField body="color" type="string">
  Color code for calendar display (e.g., "#FF5733")
</ParamField>

<ParamField body="notes" type="string">
  Additional notes
</ParamField>

<ParamField body="details" type="array">
  Array of event detail items (resources/tasks)
  <ParamField body="resource_item_id" type="integer" required>

  </ParamField>

  </ParamField>
  <ParamField body="description" type="string">
    Task/activity description
  </ParamField>
  <ParamField body="cost" type="number">
    Estimated or actual cost for this item
  </ParamField>
  <ParamField body="is_completed" type="boolean">
    Completion status (default: false)

## Update Calendar Event

<CodeGroup>
```bash PUT /api/calendar/events/{event_id}/
curl -X PUT https://api.mantis.com/api/calendar/events/567/ \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Updated Maintenance Schedule",
    "priority": "URGENT",
    "status": "IN_PROGRESS"
  }'
```

```bash PUT (Alternative)
curl -X PUT https://api.mantis.com/api/calendar/events/ \
  -H "Content-Type: application/json" \
  -d '{
    "id": 567,
    "title": "Updated Title"
  }'
```
</CodeGroup>

Update an existing calendar event. All fields except those required for creation are optional.

### Path Parameters

<ParamField path="event_id" type="integer">
  Calendar event ID (can also be provided in request body as "id")
</ParamField>

### Request Body

<ParamField body="id" type="integer">
  Event ID (if not provided in path)
</ParamField>

All other fields from the create endpoint are optional. Only provided fields will be updated.

### Updating Details

When updating the `details` array:
- Existing details are soft-deleted (marked as inactive)
- New details from the request are created
- This allows complete replacement of the details list

### Response

<ResponseField name="success" type="boolean">
  Indicates if update was successful
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable message
</ResponseField>

<ResponseField name="data" type="object">
  Updated calendar event object
</ResponseField>

## Move Event (Drag & Drop)

<CodeGroup>
```bash PATCH /api/calendar/events/{event_id}/move/
curl -X PATCH https://api.mantis.com/api/calendar/events/567/move/ \
  -H "Content-Type: application/json" \
  -d '{
    "start_date": "2026-03-25"
  }'
```

```bash PATCH (With Duration)
curl -X PATCH https://api.mantis.com/api/calendar/events/567/move/ \
  -H "Content-Type: application/json" \
  -d '{
    "start_date": "2026-03-25",
    "end_date": "2026-03-27"
  }'
```
</CodeGroup>

Move an event to a new date. If the event has an end date, the duration is preserved unless explicitly overridden.

### Path Parameters

<ParamField path="event_id" type="integer">
  Calendar event ID (can also be provided in request body as "id")
</ParamField>

### Request Body

<ParamField body="start_date" type="string" required>
  New start date (YYYY-MM-DD)
</ParamField>

<ParamField body="end_date" type="string">
  New end date (YYYY-MM-DD). If not provided and event has duration, end date is adjusted to maintain the same duration.
</ParamField>

### Duration Preservation

If the event originally spans 3 days (e.g., March 1-3) and you move it to March 10, the end date automatically becomes March 12 to preserve the 3-day duration.

### Response

<ResponseField name="success" type="boolean">
  Indicates if move was successful
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable message
</ResponseField>

<ResponseField name="data" type="object">
  Updated calendar event object
</ResponseField>

## Get Calendar Event

<CodeGroup>
```bash GET (Single)
curl -X GET https://api.mantis.com/api/calendar/events/567/
```

```bash GET (By Project)
curl -X GET "https://api.mantis.com/api/calendar/events/?project_id=789"
```

```bash GET (By Project and Month)
curl -X GET "https://api.mantis.com/api/calendar/events/?project_id=789&month=3&year=2026"
```
</CodeGroup>

Retrieve a single event or list events for a project, optionally filtered by month.

### Path Parameters (Single Event)

<ParamField path="event_id" type="integer">
  Calendar event ID
</ParamField>

### Query Parameters (List)

<ParamField query="project_id" type="integer" required>
  Project ID to filter events
</ParamField>

<ParamField query="month" type="integer">
  Month number (1-12) to filter events
</ParamField>

<ParamField query="year" type="integer">
  Year to filter events (required if month is provided)
</ParamField>

### Month Filtering Logic

Events are included if they overlap with the specified month:
- Event starts before month ends AND
- Event ends after month starts (or has no end date)

This ensures multi-day events spanning month boundaries are included.

### Response

<ResponseField name="success" type="boolean">
  Indicates if query was successful
</ResponseField>

<ResponseField name="data" type="object" or="array">
  Single event object or array of event objects
</ResponseField>

## Workflow Examples

### Schedule Maintenance Event

```bash
# 1. Create event
curl -X POST https://api.mantis.com/api/calendar/events/ \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": 789,
    "title": "Monthly Equipment Check",
    "event_type": "MAINTENANCE",
    "priority": "MEDIUM",
    "start_date": "2026-04-01",
    "start_time": "09:00",
    "responsible_technical_id": 123
  }'

# 2. Add tasks
curl -X PUT https://api.mantis.com/api/calendar/events/ \
  -H "Content-Type: application/json" \
  -d '{
    "id": 567,
    "details": [
      {"resource_item_id": 101, "description": "Check oil level", "cost": 0},
      {"resource_item_id": 102, "description": "Replace filter", "cost": 150}
    ]
  }'

# 3. Start work
curl -X PUT https://api.mantis.com/api/calendar/events/ \
  -H "Content-Type: application/json" \
  -d '{"id": 567, "status": "IN_PROGRESS"}'

# 4. Mark task complete
curl -X PUT https://api.mantis.com/api/calendar/events/ \
  -H "Content-Type: application/json" \
  -d '{
    "id": 567,
    "details": [
      {
        "resource_item_id": 101,
        "description": "Check oil level",
        "is_completed": true,
        "completed_date": "2026-04-01"
      }
    ]
  }'
```

### Get Monthly Calendar View

```python
import requests
from datetime import datetime

# Get all events for March 2026
response = requests.get(
    "https://api.mantis.com/api/calendar/events/",
    params={
        "project_id": 789,
        "month": 3,
        "year": 2026
    }
)

events = response.json()["data"]

# Group by date
from collections import defaultdict
events_by_date = defaultdict(list)

for event in events:
    date = event["start_date"]
    events_by_date[date].append(event)

# Display
for date in sorted(events_by_date.keys()):
    print(f"\n{date}:")
    for event in events_by_date[date]:
        print(f"  - {event['title']} ({event['priority_display']})")
```

### Drag and Drop Event

```javascript
// Frontend drag-and-drop handler
function onEventDrop(eventId, newStartDate) {
  fetch(`https://api.mantis.com/api/calendar/events/${eventId}/move/`, {
    method: 'PATCH',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify({
      start_date: newStartDate
    })
  })
  .then(response => response.json())
  .then(data => {
    console.log('Event moved:', data);
    // Refresh calendar view
  });
}
```
