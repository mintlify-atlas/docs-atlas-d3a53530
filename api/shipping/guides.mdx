# Shipping Guides API

Manage shipping guides (guías de remisión) for tracking equipment and material transport between locations. Supports entry and exit guides with detailed item tracking.

## Create Shipping Guide

<CodeGroup>
```bash POST /api/shipping/guides/
curl -X POST https://api.mantis.com/api/shipping/guides/ \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": 789,
    "type_shipping_guide": "EXIT",
    "issue_date": "2026-03-15",
    "origin_place": "Warehouse A",
    "destination_place": "Field Site B",
    "carrier_name": "Juan Pérez",
    "carrier_ci": "1234567890",
    "vehicle_plate": "ABC-1234",
    "details": [
      {
        "description": "Equipment Case",
        "quantity": 5,
        "unit": "units",
        "id_resource_item": 101
      }
    ]
  }'
```
</CodeGroup>

Create a new shipping guide with transport details and item manifest.

## Update Shipping Guide

<CodeGroup>
```bash PUT /api/shipping/guides/
curl -X PUT https://api.mantis.com/api/shipping/guides/ \
  -H "Content-Type: application/json" \
  -d '{
    "id": 456,
    "destination_place": "Updated Field Site C",
    "details": [
      {
        "description": "Equipment Case",
        "quantity": 10,
        "unit": "units"
      }
    ]
  }'
```
</CodeGroup>

Update an existing shipping guide. Only guides in `DRAFT` status can be edited.

### Request Body

<ParamField body="id" type="integer" required>
  Shipping guide ID to update
</ParamField>

All other fields from the create endpoint are optional. Only provided fields will be updated.

### Response

<ResponseField name="success" type="boolean">
  Indicates if update was successful
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable message
</ResponseField>

<ResponseField name="data" type="object">
  Updated shipping guide object
</ResponseField>

## Change Guide Status

<CodeGroup>
```bash PATCH /api/shipping/guides/
curl -X PATCH https://api.mantis.com/api/shipping/guides/ \
  -H "Content-Type: application/json" \
  -d '{
    "id": 456,
    "status": "CLOSED"
  }'
```
</CodeGroup>

Change the status of a shipping guide. Status transitions are validated.

### Request Body

<ParamField body="id" type="integer" required>
  Shipping guide ID
</ParamField>

<ParamField body="status" type="string" required>
  New status. Options: `DRAFT`, `CLOSED`, `VOID`
</ParamField>

### Valid Status Transitions

- From `DRAFT`: Can change to `CLOSED` or `VOID`
- From `CLOSED`: Can change to `VOID` only
- From `VOID`: No transitions allowed (final state)

### Response

<ResponseField name="success" type="boolean">
  Indicates if status change was successful
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable message
</ResponseField>

<ResponseField name="data" type="object">
  Updated shipping guide object
</ResponseField>

<ResponseExample>
```json Error - Invalid Transition
{
  "success": false,
  "error": "No se puede cambiar de Cerrado a Borrador. Transiciones permitidas: ['VOID']"
}
```
</ResponseExample>

## Get Shipping Guide

<CodeGroup>
```bash GET (Single)
curl -X GET https://api.mantis.com/api/shipping/guides/456/
```

```bash GET (By Project)
curl -X GET "https://api.mantis.com/api/shipping/guides/?project_id=789"
```
</CodeGroup>

Retrieve a single shipping guide by ID or list all guides for a project.

### Path Parameters (Single Guide)

<ParamField path="guide_id" type="integer">
  Shipping guide ID
</ParamField>

### Query Parameters (List)

<ParamField query="project_id" type="integer" required>
  Project ID to filter guides
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if query was successful
</ResponseField>

<ResponseField name="data" type="object" or="array">
  Single guide object or array of guide objects
</ResponseField>

## Workflow Example

### 1. Create Draft Guide

```bash
curl -X POST https://api.mantis.com/api/shipping/guides/ \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": 789,
    "issue_date": "2026-03-15",
    "type_shipping_guide": "EXIT",
    "origin_place": "Warehouse",
    "destination_place": "Field Site",
    "carrier_name": "Transport Co.",
    "vehicle_plate": "XYZ-789"
  }'
```

### 2. Add/Update Details

```bash
curl -X PUT https://api.mantis.com/api/shipping/guides/ \
  -H "Content-Type: application/json" \
  -d '{
    "id": 456,
    "details": [
      {"description": "Equipment A", "quantity": 5, "unit": "units"},
      {"description": "Equipment B", "quantity": 3, "unit": "units"}
    ]
  }'
```

### 3. Close Guide

```bash
curl -X PATCH https://api.mantis.com/api/shipping/guides/ \
  -H "Content-Type: application/json" \
  -d '{"id": 456, "status": "CLOSED"}'
```

### 4. Upload PDF

```bash
curl -X POST https://api.mantis.com/api/load_files/ \
  -F "model_type=shipping_guide" \
  -F "object_id=456" \
  -F "field_name=shipping_guide_file" \
  -F "file=@guide.pdf"
```
